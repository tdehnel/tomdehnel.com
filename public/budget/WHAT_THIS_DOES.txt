============================================================
WHAT THIS DOES — Household Budget Questionnaire
============================================================

------------------------------------------------------------
SECTION 1: PLAIN-LANGUAGE SUMMARY
------------------------------------------------------------

- This is a private questionnaire that helps one person map out a
  household budget for themselves.

- You go through a long list of spending categories (housing, food,
  travel, and so on) and, for each one, you say two numbers:
    * the LEAST you'd want to spend if money got tight (the "minimum"), and
    * the amount that feels COMFORTABLE.

- It also asks some "how do you feel about money" questions — things
  like how much you want to save, how willing you are to cut back, and
  what matters most to you.

- When you're done, you can save your answers to a file ("Export for
  Claude"). You can hand that file to Claude and ask for an analysis.
  If a second person fills out their own copy and exports it too, you
  can give Claude both files and ask it to compare the two.

- Everything you type stays inside this one web browser, on this one
  computer. Nothing is sent anywhere, there is no login, and nothing
  is saved to any website or server. If you clear the browser data,
  the answers are gone.

- It's meant for PERSONAL household spending only — not business
  expenses, rental income, or rental-property costs.


------------------------------------------------------------
SECTION 2: TECHNICAL DESCRIPTION
------------------------------------------------------------

INPUTS
- One budget profile per browser, with an optional name used only to
  label the export file.
- For each spending "leaf" category: a monthly Minimum dollar amount, a
  monthly Comfortable dollar amount, and an optional free-text note.
- A set of attitude/nuance answers (sliders, radio choices, a
  willingness-to-cut rating per category, a list of big upcoming
  expenses, and free-text reflections).

PROCESSING / TRANSFORMATIONS
- Answers are written to the browser's localStorage on every keystroke
  (debounced), under the single key "budgetApp.profile", with a
  schemaVersion field for future migrations. (Data from an older
  two-profile build, if present under "budgetApp.profile.tom/.wife", is
  adopted once so nothing already entered is lost.)
- The summary sums each group and the whole budget, and shows the
  annualized figure (monthly x 12).
- Real-time form validation runs on each amount field: amounts cannot
  be negative (native min="0"), and the Comfortable amount must be at
  least the Minimum (enforced with the Constraint Validation API).
- "Export for Claude" generates a single self-describing Markdown
  document: a short instruction header for Claude, then the person's
  totals, a per-category table, the answered line items (with notes),
  and their attitude answers. The header also tells Claude that if a
  second person's budget is supplied in the same conversation, it
  should compare the two.

OUTPUTS
- On-screen questionnaire and per-person summary.
- A downloaded Markdown file (household-budget-[name-]YYYY-MM-DD.md)
  intended to be uploaded to Claude for analysis.
- A print / "Save as PDF" view (via the browser print dialog and a
  dedicated print stylesheet). Apart from the export download and
  printing, no files are written to disk by the app itself; only the
  browser's own localStorage is used.

DEPENDENCIES
- NONE. This is a pure static app: plain HTML, CSS, and vanilla
  JavaScript with no framework, no build step of its own, no npm
  packages, no external CDN, and no network calls. For that reason
  there is intentionally no package.json / requirements.txt in this
  folder — there is nothing to install. It is deployed simply by being
  copied into the parent Astro site's build output (public/ -> dist/)
  and served at tomdehnel.com/budget/.
